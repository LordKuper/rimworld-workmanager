LangVersion=latest resolves the C# language version from the Roslyn compiler in the build SDK (10.0.300 → C# 14), not from the net472 target. Code compiles modern C# syntax but runs on RimWorld's fixed Mono/Unity (net472 BCL); features needing newer BCL/runtime support require polyfill shims or are unavailable.

Build discipline (WorkManager.csproj): TreatWarningsAsErrors=true + WarningLevel 9999 (Debug and Release), Nullable=enable, GenerateDocumentationFile=true. A warning fails the build.

Constraints

  • Target framework pinned — net472 is fixed by RimWorld's Mono/Unity runtime. Do not upgrade the TFM.
  • Game version range pinned — supports RimWorld 1.1–1.6; runtime API surface is whatever the loaded game version exposes.
  • No bundled runtime deps — Harmony and LordKuper.Common are referenced compile-only / reference-only (Private=False, ExcludeAssets=runtime) and provided at runtime by their own mods — must never be redistributed inside this mod.
  • Zero-warning build — TreatWarningsAsErrors=true, WarningLevel 9999; any warning fails the build.
  • Nullable enabled — Nullable=enable across Source and Tests; the nullable context must not be disabled.
  • FluentAssertions license guard — FluentAssertions must stay on 7.x (Apache-2.0); 8.x is commercial-licensed and forbidden.
  • Mod license — CC BY-NC-SA 4.0: share-alike, attribution, non-commercial.

Architecture Principles

  • Compile against, never ship, the host — RimWorld Assembly-CSharp, Unity modules, Harmony and LordKuper.Common are referenced with Private=False / compile-only assets. Keeps the payload small and avoids assembly conflicts; runtime copies come from the game and dependency mods.
  • Patch, don't fork the game — behaviour is injected via Harmony runtime patching rather than replacing game code — aligns with the concept's "not a work-tab UI replacement" / "plays well with others" pillars.
  • Defer to other mods where present — loadAfter ordering plus compatibility shims; integrate via others' APIs instead of conflicting.
  • Build on the shared parent library — common helpers/abstractions live in LordKuper.Common; WorkManager consumes them (RW 1.6 dependency) rather than re-implementing, and inherits its coding standards.
  • Strict, self-contained code — zero-warning build + nullable + XML docs; code must be self-sufficient without ASD design docs. Quality enforced by the toolchain, not convention.
  • Multi-version support via metadata + conditional deps — About.xml supportedVersions 1.1–1.6 and modDependenciesByVersion (Common only on 1.6) keep one codebase serving multiple game versions.
